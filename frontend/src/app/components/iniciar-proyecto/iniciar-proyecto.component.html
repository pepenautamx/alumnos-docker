🔑 Paso 2: Crear el archivo de Variables de Entorno

El backend necesita leer las credenciales de conexión. Crea un archivo de texto llamado .env dentro de la carpeta /backend:

DB_HOST=localhost
DB_PORT=5435

💡 Nota: Si vas a correr todo usando Docker Compose, la variable DB_HOST debe cambiarse de localhost a db (que es el nombre del servicio de la BD en la red virtual).

🚀 Opción A: Despliegue Rápido con Docker (Recomendado)

Esta es la vía más rápida y limpia. No necesitas instalar paquetes de Node ni inyectar bases de datos de forma manual. Docker compilará el backend, creará el servidor Nginx para el frontend e inicializará las tablas con un solo comando:

Ejecuta este comando en la **raíz del proyecto** (donde está el archivo docker-compose.yml):

docker compose up -d --build

🌐 Direcciones de acceso locales en Docker:

  • Frontend (Angular + Nginx): Abre http://localhost (Puerto 80).
  • Backend (Apollo GraphQL Playground): Abre http://localhost:4000.
  • Base de Datos: Mapeada externamente en el puerto 5435.

💡 El semillado y creación de tablas es 100% automático gracias al script mapeado de /db-init/init.sql.

💻 Opción B: Modo Desarrollo Local (Para programar código)

Si tu intención es modificar y programar código en tiempo real con recarga automática, debes levantar los servicios localmente paso a paso:

Paso B1. Encender la base de datos (Docker):

docker run --name db_alumnos -e POSTGRES_USER=pepe_admin -e POSTGRES_PASSWORD=supersecretpassword -e POSTGRES_DB=escuela_db -p 5435:5432 -d postgres:15-alpine

(Recuerda conectarte mediante DBeaver y ejecutar el script SQL de creación de tablas de la Fase 1 o usar init.sql).

Paso B2. Arrancar el Backend (GraphQL):

cd backend
npm install
node index.js

El servidor levantará en http://localhost:4000.

Paso B3. Arrancar el Frontend (Angular):

cd frontend
npm install --legacy-peer-deps
npx ng serve

El frontend de desarrollo levantará en http://localhost:4200.

🕵️ Verificación y Solución de problemas rápidos

¿Algo no responde al levantar con Docker Compose? Ejecuta estos comandos de diagnóstico:

  • Verificar estado de contenedores: docker compose ps (Asegúrate de que digan "Up").
  • Ver logs en tiempo real: docker compose logs -f (Busca errores de conexión a Postgres).
  • Reiniciar desde cero: docker compose down && docker compose up -d --build.
